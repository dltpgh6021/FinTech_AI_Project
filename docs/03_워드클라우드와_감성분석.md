# Day 3. 워드클라우드와 감성분석

 > TIL (Today I Learned)
 > - 수집된 텍스트 데이터의 시각화(WordCloud)
 > - NLP 딥러닝 모델을 활용한 금융 뉴스 긍정/부정 감성 분석 (Sentiment Analysis)
 > - 주피터 노트북 코드를 'src/' 디렉토리의 파이썬 모듈('.py')로 리팩토링

 ## 사용한 기술 스택 (Tech Stack)
- **Data Manipulation:** `pandas`
- **Visualization:** `wordcloud`, `matplotlib`
- **AI / NLP:** `transformers` (Hugging Face), `Pytorch`
- **Model:** `snunlp/KR-FinBert-SC` (서울대학교 한국어 금융 뉴스 감성 분석 모델)

## 핵심 학습 내용 및 성과

### 1. WordCloud 시각화
- 수집된 뉴스 제목 텍스트를 병합하여 자주 등장하는 키워드를 직관적으로 파악. 
- 한글 폰트 깨짐을 방지하기 위해 로컬 환경의 맑은 고딕 폰트 경로를 명시

### 2. AI 감성 분석 (Sentiment Analysis) 도입
- Hugging Face의 `pipeline`을 활용하여 금융 도메인에 특화된 `KR-FinBert-SC` 모델 로드. 
- 크롤링한 뉴스 기사 제목을 모델에 입력 -> 해당 기사가 주가에 미칠 영향을 긍정/부정/중립으로 자동분류 & 확신도(Score)를 DataFrame에 추가
- *인사이트: * '급락', '매도 폭탄' 등 직관적인 단어는 99% 이상의 확신도로 부정 판별해냄. 하지만 '7%↓', '사이드카 발동' 등 특수기호 혹은 전문 금융 용어 위주의 기사는 중립으로 판별하는 모델의 한계점(Edge Case)을 확인, 추후 도메인 사전 추가 등 전처리 로직 개선의 여지를 발견함. 

### 3. 코드 모듈화 및 아키텍쳐 개선 (Refactoring)
- 파편화된 주피터 노트북 실험 코드를 재사용 가능한 함수 단위로 묶어 분리. 
- `if __name__ == "__main__":` 구조를 활용하여 모듈의 독립적인 테스트 환경 구축 및 폴더 간 상대 경로 참조 로직 적용

## 에러와 해결 과정
